## Deliberately excluded (CP-territory, low ROI for ML interviews)
- Segment trees, Fenwick trees, tries (except if asked for autocomplete-style system design)
- Heavy number theory (modular exponentiation, sieve variants, combinatorics beyond basics)
- Advanced string algorithms (KMP, Z-function, suffix arrays)
- Game theory / Nim-style problems
- Bitmask DP, digit DP

## Suggested pace
- 5–6 problems/week alongside your ML/HOML work → ~150 problems in ~26–28 weeks
- If compressing for October: 8–10/week → done in ~16 weeks, matching your existing DSA plan timeline
- Always redo Two Sum, Merge Intervals, Kth Largest, and Course Schedule cold — these 4 patterns cover a disproportionate share of actual interview questions
- Track weak patterns separately (yours: heaps, intervals) and over-index practice there
